# Genesis Bots

**Skill version: 2026.06.23** · [how to update](#updating-this-skill)

Call the Genesis (OpenClaw) bot API. The bots are OpenAI-compatible chat models where the
`model` field is a **bot slug**. Each bot carries its own server-side system prompt — you send
user messages, the bot does the rest.

## Genesis vs Exodus — am I the right skill?

You're looking at **Genesis**: the manual, hands-on way to use the copywriting server. You call
individual bots directly with the member's own `gen_` key and your own prompts, and you drive
every step yourself.

There's a done-for-you sibling called **Exodus** — pre-built `npx @aicopycoders/exodus` pipelines (the
`exodus-*` skills) that drive these *same* Genesis bots automatically against the member's brand.
Think of it as a car: **Genesis is the manual transmission** (you shift every gear yourself),
**Exodus is the automatic** (same engine underneath, it does the driving). Most people who don't
know the internals just want their ads — your job is to get them into the right one.

**Exodus may or may not be installed here** — plenty of members run Genesis on its own. So check
`.claude/skills/` for `exodus-*` skills before assuming:

- **No `exodus-*` skills present** → there's only one system and you're it. Carry on; ignore the
  rest of this section.
- **`exodus-*` skills present** → both are installed. Route by what the user said: bare
  **"genesis" / "run the bots" / "openclaw" / a bot slug** → this skill; **"exodus …"** → the
  `exodus-*` skills (hand off, don't do their pipeline work here). If the user just wants "ads"
  and clearly doesn't know the difference, ask one question — *"Do you want to drive the Genesis
  bots directly, or run an Exodus pipeline?"* — rather than guessing, because the two go down
  completely different paths.

## Setup (one time, ~1 minute)

You need two keys. Get them from the member portal — it walks you through it:

- **Portal:** https://genesis.copycoders.ai/portal — your Genesis key (`gen_...`) is already there
- **Setup Guide:** https://genesis.copycoders.ai/setup — how to get your Anthropic key (`sk-ant-...`)

Then put both keys in a local `.env` file — never paste them into code or chat:

```bash
cp .env.example .env      # 1. make your private copy
# 2. open .env, paste your two keys, save
```

You only fill in two values (the API address is pre-filled — leave it alone):

| In `.env` | What it is | Where to get it |
|-----------|------------|-----------------|
| `GENESIS_API_KEY` | Your Genesis access key (`gen_...`) | the portal (already generated for you) |
| `ANTHROPIC_API_KEY` *or* `OPENROUTER_API_KEY` | Your own provider key — how you pay for usage | the Setup Guide |

Your provider key can be an Anthropic key (`sk-ant-...`, from console.anthropic.com) **or** an
OpenRouter key (`sk-or-...`, from openrouter.ai):

- **Want every model** — Claude *and* GPT-5, Gemini, etc.? Use an **OpenRouter key** (one key
  serves them all) and put it in `OPENROUTER_API_KEY`.
- **Only need Claude?** An Anthropic key is enough — put it in `ANTHROPIC_API_KEY`.

You can set **both**: the helper automatically picks the right key per request — Claude bills
direct to Anthropic (cheaper), while GPT-5/Gemini route via OpenRouter. Keys are detected by
their `sk-ant-`/`sk-or-` prefix, so it still works even if you paste one into the other field.

The bundled helper at `.claude/skills/genesis-bots/scripts/genesis-stream.mjs` (Node — works everywhere, no Python needed; a `genesis-stream.py` is also bundled as a fallback) **reads `.env`
automatically** — you don't need to know what an environment variable is or run any
`export`/`source` commands. Just edit `.env` once.

> These same names (`GENESIS_API_KEY`, `ANTHROPIC_API_KEY`) and base URL match the portal exactly,
> so if you're configuring this inside OpenClaw instead of a `.env` file, use the identical values.
> Keep `.env` private. Add it to `.gitignore`; commit only `.env.example`.

For the raw `curl` commands below (optional), load the file into your shell first:
`set -a && source .env && set +a`

## Updating this skill

Genesis adds features to this skill over time (the version is at the top of this file).
Updating an existing install needs **no portal login**:

- **Easiest — ask your agent:** *"update the genesis-bots skill."* It downloads the latest
  bundle and replaces this `SKILL.md` and the helper script in place, leaving your `.env`
  untouched.
- **By hand (docs only, one line)** — overwrites just `SKILL.md`, never your `.env`:
  ```bash
  curl -fsSL https://genesis.copycoders.ai/downloads/genesis-bots-SKILL.md \
    -o .claude/skills/genesis-bots/SKILL.md
  ```
  For a full update including the helper script, re-download
  `https://genesis.copycoders.ai/downloads/genesis-bots-skill.zip` instead.

Either way, **start a new session** afterward (skills load only at session start; `/clear` works).

## Discover bots (never hardcode slugs)

The catalog changes — always pull the live list. Categories include Copywriting, Research,
Strategy, Creative, Image Prompts, Pages & Funnels, Video Pipelines, and Other.

```bash
curl -s -H "Authorization: Bearer $GENESIS_API_KEY" \
  "$GENESIS_BASE_URL/models" \
  | jq -r '.data[] | "\(.id)\t\(._genesis.category)\t\(._genesis.name)"'
```

Filter by category: append `?category=Copywriting`. Each entry's `id` is the slug you pass as
`model`; `_genesis` holds `{name, description, category, provider, model}`.

## Call a bot (streaming helper)

The bundled helper streams the response, accumulates it, and writes it to a file. Use the Node
helper (`genesis-stream.mjs`) — Claude Code always has Node, and it needs nothing installed. It
lives inside this skill folder — call it with its full path from your project root (the relative
`scripts/...` form only works if you've `cd`'d into the skill folder first):

```bash
# 1. write the prompt
cat > prompt.txt << 'EOF'
{your context + ask}
EOF

# 2. call: <bot-slug> <prompt-file> <output-file> [temperature]
node .claude/skills/genesis-bots/scripts/genesis-stream.mjs BOT_SLUG prompt.txt out.md

# (No Node? A Python fallback ships alongside it — identical arguments:)
# python3 .claude/skills/genesis-bots/scripts/genesis-stream.py BOT_SLUG prompt.txt out.md
```

Request fields:

| Field | Notes |
|-------|-------|
| `model` | the **bot slug**, not an LLM model name. Append `@<model>` to run the bot on a different LLM — see [Choose the model and temperature](#choose-the-model-and-temperature-optional). |
| `messages` | `role` must be `user` or `assistant`. **`system` messages are dropped** — the bot's prompt is server-side. |
| `stream` | default `true`; SSE in OpenAI format. Set `false` for a complete JSON body. |
| `temperature` | optional creativity dial; omit to use the bot's default. Ignored by Opus 4.7+/Fable — see [Choose the model and temperature](#choose-the-model-and-temperature-optional). |
| `max_tokens` | optional; bot has a default. |

### Choose the model and temperature (optional)

Two optional, per-request dials. Set them when the user wants them; otherwise leave the bot's
defaults alone (the normal case — nothing to do).

**Model** — set `model` to `<bot-slug>@<model>` (everything after the first `@` is the model;
no `@` = the bot's default). Which models you can pick depends on the provider key in `.env`:

| Your key | Models you can request | Examples |
|----------|------------------------|----------|
| Anthropic (`sk-ant-…`) | Claude only (`claude-*`, no `/`) | `claude-opus-4-8` (smartest), `claude-sonnet-4-6`, `claude-haiku-4-5` (fast/cheap) |
| OpenRouter (`sk-or-…`) | any model | `openai/gpt-5`, `google/gemini-2.5-pro`, plus all Claude models |

A non-Claude model on an Anthropic key returns `400`. A bot's current default is the
`_genesis.model` field in the `/models` list.

**Temperature** — the creativity dial: higher (e.g. `0.9`) = more varied/creative, lower
(e.g. `0.2`) = more focused/consistent. Pass it as the helper's 4th argument (or the
`temperature` field). Omit it to use the bot's tuned default.

**Handle model/temperature compatibility for the user — don't make them think about it.**
Temperature works on Opus 4.6 and earlier, Sonnet, Haiku, and OpenRouter models. The newest
reasoning models — **Opus 4.7+ and Fable — ignore temperature** (that control was removed). So:

- User wants a custom temperature → use a model that supports it (the bot's default already
  does); don't pair it with a model that can't.
- User wants the most powerful model (Opus 4.8) → just switch the model; don't set a temperature.
- User asks for both at once → tell them briefly they can have a custom temperature **or** the
  newest model, and pick whichever better fits their goal. Never make them debug a silent no-op.

```bash
# model + temperature (4th arg = temperature) — opus-4-6 supports it:
node .claude/skills/genesis-bots/scripts/genesis-stream.mjs BOT_SLUG@claude-opus-4-6 prompt.txt out.md 0.9

# most powerful model, no temperature (ignored there anyway):
node .claude/skills/genesis-bots/scripts/genesis-stream.mjs BOT_SLUG@claude-opus-4-8 prompt.txt out.md

# raw curl with both fields:
curl -s -X POST "$GENESIS_BASE_URL/chat/completions" \
  -H "Authorization: Bearer $GENESIS_API_KEY" \
  -H "X-Provider-Key: $ANTHROPIC_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"BOT_SLUG@claude-opus-4-6","temperature":0.9,"messages":[{"role":"user","content":"..."}],"stream":false}'
```

### Chaining bots

Feed one bot's output into the next as context in the user message (e.g. buyer profile →
creative brief → hooks → ads). The API injects each bot's system prompt for you; you only pass
messages. Use the slugs from the live models list.

## Limits

- **1 concurrent stream per key** — a 2nd parallel stream on the same key returns
  `429 connection_limit_error`. Call sequentially per key.
- **60 requests/min per key** (sliding window) — over the limit returns `429` with `Retry-After`.
- Long generations are fine: the server sends SSE heartbeats (~every 5s) to prevent proxy timeouts.

## Minimal call without the helper

```bash
curl -s -X POST "$GENESIS_BASE_URL/chat/completions" \
  -H "Authorization: Bearer $GENESIS_API_KEY" \
  -H "X-Provider-Key: $ANTHROPIC_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"BOT_SLUG","messages":[{"role":"user","content":"..."}],"stream":false}' \
  | jq -r '.choices[0].message.content'
```

## Error codes

- `400` — missing `X-Provider-Key` or malformed request; also returned when a `@<model>` override
  isn't allowed for your key (e.g. a non-Claude model with an `sk-ant-…` key) — the message says so
- `401` — invalid/revoked Genesis key
- `404` — bot slug not found (re-check the models list)
- `429` — rate limit or concurrent-stream limit hit
- `500` — provider error (usually a bad/over-quota provider key)
