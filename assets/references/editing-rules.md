# Editing Rules — the voice, the steer-prompts, the finish

> Part of the creative-strategy framework set; compact index lives in `exodus-strategist`.
> Editing is a first-class stage of the wheel, not an afterthought: how you push an ad until
> it's right and clean it to a human voice. In Exodus, the editing pass is built into the
> `genesis` run as a per-variant menu — and every good edit you make should bake back into the
> brand primer so the next run inherits it.

## The steer-prompts (drop-in directions)
These are the named moves you steer a draft with. The first five map directly to the Genesis
per-variant editing menu (Natural Language / Shorten / Cut / Simplify / Make Better); the rest
are the same kind of small, repeatable direction.
- **Natural language / anti-AI** — too AI → natural; too staccato → natural; short clipped
  sentences → natural.
- **More bounce** — conversational, musical flow.
- **Shorten (to a word target)** — read through and **cut specific lines**; do NOT compress
  sentences into "little turds."
- **Cut** — be anal; remove fluff lines that don't move the argument forward.
- **Simplify (reading level)** — swap high-falutin Latin words for concrete, earthy
  Germanic-rooted words; ~8th grade → 3–5th grade. *Not* dumber, *not* less intense — just
  brute-simple. (This is the "caveman copy" bar.)
- **Make better** — "weave" in (don't bolt on) proof / logic / insight; concrete visual
  language; intensify the pain or the benefit so they *feel* it. (Name the Copy Block —
  `copy-blocks.md` — instead of saying "better.")
- **Fix transitions** — add "and / because / so" connective tissue.
- **Fix logic** — does this make sense; stress-test specific ideas.
- **CTA** — explicit (formula below) + clear characterizations.
- **Find & replace (semantic)** — e.g. "20 seconds" → "30 seconds" across all ads.

## Core sentence rules
- **One sentence per line. One idea per line.** Each line stands alone; break separate ideas
  apart (keep as one line only if tightly connected and flowing).
- **No staccato fragment phrasing** ("No this. No that." / "Not X — it's Y") as a rhythm
  trick. Only break for rhythm if it genuinely serves the story.
- **Write ACTUAL sentences, not fragments.** If it wouldn't make sense read aloud, fix it
  ("Empty." → "It's empty.").
- **Bounce + natural transitions** — break compound thoughts so each idea breathes, guide
  between them with And / Because / So / But. Reads like someone telling a story, not a list.

## Story structure
- **Tell it linearly.** Open on a specific night / specific moment and unfold in real time.
  Don't open vague or jump around in time.
- **Story FIRST, explanation SECOND** — let the emotional beat land before pulling back to
  the research/mechanism.
- **Don't cite specific years** — "researchers found" / "a study showed." Keep it timeless.

## Endings & CTA
End on **intrigue** with a fun, relatable characterization of the solution (feels like a
discovery, not a pitch) — "a little 30-second thing I do every morning with my coffee." Then
support an explicit CTA.
**CTA formula:** *[explicit action phrase] + [casual characterization] + [brief benefit
tease, 2–4 lines: mechanism/transformation] + [personal proof payoff: first-person, specific,
before→after].*

## Anti-AI — avoid
- *Structural:* "statement: then list"; balanced "This isn't X, it's Y"; three-part lists;
  formal transitions (However, Moreover, Indeed, Subsequently, Furthermore, Additionally).
- *Red-flag phrases:* "Not just X, but Y," "Whether X or Y," "This means that…," "Let's
  explore/dive into," "Picture this," "The key/truth is," jargon (leverage, drive results,
  optimize), intensifiers (ultimately, essentially, simply), "One might argue…," "It is worth
  noting…"
- *AI words:* hurdles, harnessing, unveil, realm, delve, dive in, embark, navigate, tapestry,
  craft, dazzle, evoke, transform, profound, game-changing, insights, synergy, leverage.
- *Embrace instead:* varied paragraph lengths; fragments-as-speech starting And/But/So;
  conversational asides; small imperfections; clear opinions without hedging; pattern
  interrupts; spontaneous questions; big sentence-length variation; specific personal details;
  conversational flow over perfect structure.

## The finish
Re-hook (new hooks from the hook primer + the edited ad) → select → headlines (from the
headline primer) → select → format into a clean doc → on to Creative.

## Feedback bakes back (the compounding loop)
This is what makes Editing compound instead of repeat. Two loops:
1. **Repeated edits → reusable steer-prompts.** When you find yourself giving the same
   correction across ads, it's no longer a one-off — it's a button. Capture it so it becomes
   a direction you press, not a sentence you retype.
2. **Feedback → primer rules.** The corrections and dos/don'ts you give a draft get written
   back into the brand primer (tag them so they're visible, e.g. `[banked YYYY-MM-DD]`). The
   next `genesis` run reads the updated primer and inherits the fix. A correction you make
   once should never have to be made the same way twice.
