# Iterations — CASHED (variations · scaling on winners)

> Part of the creative-strategy framework set; compact index lives in `exodus-strategist`.
> When an ad wins, you don't just make more random ads — you decide which of three tracks to
> run and, for variations, **turn exactly one knob.** This is the Iterations stage of the
> wheel; it's where most of the budget actually lives.

Three tracks for a winner:
- **Net-new** — a new core idea. The priority, and what keeps the account from fatiguing.
- **Variations** — freeze the winner, turn ONE knob (CASHED, below).
- **Scaling** — amplify, change nothing.

---

## CASHED — the knobs (turn exactly one)
- **C — Concept/Claim** — what you assert (the idea + its facts, proof, mechanism)
- **A — Angle** — the psychological frame you wrap the claim in
- **S — Style** — visual style / medium / who delivers it
- **H — Hook** — the opening
- **E — Edit** — tone & format dials (length, valence/intensity, register, pacing, CTA)
- **D — Demographic/Who** — the whole segment (the biggest swing — it borders net-new)

**Decision test — "smallest box that changed?"** Go big→small, stop at the first yes:
new WHO → **D** · claim/mechanism changed → **C** · same claim, new FRAME → **A** · same
claim+frame, new LOOK/presenter → **S** · same everything, new OPENING → **H** · only
tone/length/pacing → **E**. Naming the smallest box keeps a "variation" from quietly becoming
five uncontrolled changes you can't read the result of.

## IDENTIFY first — decode the winner before turning a knob
Read off what the winner is currently *set to*, then change only one thing:
Desire → D · Avatar → D · Awareness → D · Problem/Positioning → C · Mechanism → C ·
Authority/Proof → C · Emotion → E · Frame → A.

## The high-priority moves (🔴)
- **Re-Aim (D):** awareness-level recut · mechanism-segment swap.
- **Re-Concept (C):** mechanism re-explain · proof/credibility swap · overcome objections
  (pulled from comments, woven in) · extended problem section.
- **Re-Angle (A):** reframe the claim · negative framing ("honestly, don't buy this…").
- **Re-Skin (S):** swap who delivers · VO ↔ text-on-screen · organic/ugly edit · port format
  (video↔static, static extraction, native static, static + congruent landing page) · VSL
  expansion · **change setting**. For video, **use CAST** (`cast-video.md`) to pick the knob —
  push Conceit/Style for novelty, swap Actor/Terrain to scale.
- **Re-Hook (H):** volume hook-swap (proven body + 5–10 new openings — Genesis Hook Bot is
  built for exactly this) · winning-hook remake · comment-reply hook.
- **Edit (E):** tone/length/pacing dials — usually ride along inside a bigger turn rather than
  shipping alone.

## Visual list — every creative type (the Re-Skin source)
- **static · native** (unaware/problem) — built via **SCRAWLS** (`scrawls.md`); render with
  the `creative` engines (native / copy-derived) front-doored by `image`.
- **static · templated** (solution/product) — man-holding-sign · writing-on-body · meme ·
  text-overlay · chat-screenshot · review-screenshot · comparison-chart. Spread copy across
  these structured formats with `template`.
- **static · branded** (product/most) — hero product · offer/promo · packaging · before/after.
- **video · human-led** — UGC · talking-head · whiteboard · green-screen · podcast/interview ·
  street interview · founder/authority (Actor is the swappable knob).
- **video · animation-led** — Pixar/3D · claymation · motion-graphics.
- **video · presenter-less** — VO + b-roll · text-on-screen only · product demo /
  screen-recording.

  *(Video execution is roadmap in the base CLI; the list is here so you can choose a Re-Skin
  knob and brief it. Statics ship today via `image` → `creative`/`template`.)*

## Scaling (change nothing)
Budget ladder ($100 → 200 → 350 → 550 → 750 → 1,000 — don't wait) · creative-volume flood
(10–20 diverse creatives) · stack-winners mega-ad.

## The split (adjustable)
Budget ≈ **70 scale / 20 iterate / 10 net-new.** Production effort rides account maturity:
cold ≈ 85% net-new → mature ≈ 25–30% net-new (never zero — winners fatigue). The real driver
is the Strategy Map (`segments.md`): big gaps → bias net-new · well-covered + fresh winners →
bias iterate. This is the loopback that makes the system compound — every winner feeds the
next batch, and every cycle the read gets sharper.
